# Medical Q&A Based on MiniMind3

English | [中文](README.md)

This project is a small language model training experiment based on the MiniMind3 training pipeline. Instead of only calling an existing large model API, it reproduces and documents a compact workflow from pretraining, full supervised fine-tuning, GRPO-RLAIF, and medical-domain LoRA adaptation.

> Disclaimer: This project is for learning and experimental demonstration only. Medical responses generated by the model must not be used as a substitute for professional diagnosis, prescription, emergency care, or clinical advice.

## Goals

- Build a reproducible MiniMind3-style training workflow.
- Train a base model with pretraining data to obtain basic language modeling ability.
- Apply full supervised fine-tuning to improve instruction following and dialogue behavior.
- Apply GRPO-RLAIF for preference optimization with reward-model feedback.
- Apply medical LoRA fine-tuning on top of the GRPO-RLAIF checkpoint for lightweight domain adaptation.
- Compare responses across `Pretrain`, `Full SFT`, `GRPO-RLAIF`, and `GRPO-RLAIF + Medical LoRA`.

## Model Architecture

The main experiment uses the dense MiniMind Transformer architecture. The model includes a tokenizer, input embedding, multiple Transformer layers, RMSNorm, a linear output head, and softmax. Each Transformer block contains GQA attention and an FFN module.

![MiniMind Dense Model](images/LLM-structure.jpg)

MiniMind also provides an MoE architecture with a router and routed experts. This project currently uses the dense version for the main training pipeline. The MoE diagram is included for architectural reference.

![MiniMind MoE Model](images/LLM-structure-moe.jpg)

## Training Stages

### 1. Pretraining

The pretraining stage optimizes next-token prediction on text data. Its main goal is to teach the model basic language modeling and continuation ability.

The cloud pretraining checkpoint used in this project is named:

```text
pretrain_full_4090_seq380_bs32_acc4_768.pth
```

### 2. Full Supervised Fine-Tuning

The full SFT stage starts from the pretrained checkpoint and updates all model parameters. It teaches the model chat formatting, instruction following, and question answering behavior.

The full SFT checkpoint used in this project is named:

```text
full_sft_4090_seq512_bs16_acc8_768.pth
```

### 3. GRPO-RLAIF Preference Optimization

The GRPO-RLAIF stage uses the Full SFT model as the base model. The model generates multiple answers for each prompt, a reward model and rule-based rewards score the generated answers, and GRPO updates the model toward reward-preferred behavior.

The GRPO-RLAIF checkpoint is named:

```text
grpo_rlaif_from_full_sft_768.pth
```

### 4. Medical LoRA Fine-Tuning

The medical LoRA stage uses the GRPO-RLAIF model as the base model and trains only LoRA adapter parameters. It is intended to adapt the model to medical Q&A style, terminology, and answer structure.

The medical LoRA checkpoint is named:

```text
lora_medical_from_grpo_rlaif_768.pth
```

For inference with the medical adapter, load:

```text
GRPO-RLAIF base + Medical LoRA
```

## Datasets

The datasets used in the experiments come from the public MiniMind data sources and ModelScope / Hugging Face mirrors. Large `.jsonl` datasets and `.pth` model checkpoints are not stored in this repository.

Datasets used locally include:

- `pretrain_t2t.jsonl`
- `sft_t2t.jsonl`
- `lora_medical.jsonl`
- `dpo.jsonl`
- `rlaif.jsonl`
- `agent_rl.jsonl`
- `agent_rl_math.jsonl`

The repository `.gitignore` excludes:

```text
dataset/*.jsonl
out/
checkpoints/
```

## Inference Examples

Pretrained model only:

```bash
python eval_llm.py \
  --load_from ./model \
  --weight pretrain_full_4090_seq380_bs32_acc4 \
  --max_new_tokens 256
```

Full SFT:

```bash
python eval_llm.py \
  --load_from ./model \
  --weight full_sft_4090_seq512_bs16_acc8 \
  --max_new_tokens 256
```

GRPO-RLAIF only:

```bash
python eval_llm.py \
  --load_from ./model \
  --weight grpo_rlaif_from_full_sft \
  --max_new_tokens 256
```

GRPO-RLAIF + Medical LoRA:

```bash
python eval_llm.py \
  --load_from ./model \
  --weight grpo_rlaif_from_full_sft \
  --lora_weight lora_medical_from_grpo_rlaif \
  --max_new_tokens 256
```

## Evaluation

Use the same prompt set to compare the three model stages:

```text
Pretrain
Full SFT
GRPO-RLAIF
GRPO-RLAIF + Medical LoRA
```

Suggested evaluation criteria:

- Does the model understand the user question?
- Does it follow instructions?
- Is the answer structured?
- Does it include safety reminders for medical questions?
- Does it avoid overconfident diagnosis, fabrication, or unsafe advice?

The goal of the medical LoRA stage is not to replace medical professionals, but to demonstrate lightweight domain adaptation in wording, structure, and terminology.
